Software installation problem

Each module lives on the web, and can be installed on demand, but also the dependencies are all known or determined, and the system can transparently install a system by just making a persistent cache of everything. Do this to (a) OWL ontologies and linked metainformation; (b) Sets of rule files; (c) debian packages (d) python packages -- with a little tweaking of the insides of the python package system. G

Building Secure systems from scratch

One goal here is to build a system without relying on external agencies to provide key infrastrurcture, authentication, so that the whole system is expressed in N3, and proofs can therefore be don..

An alternative goal is to import an existing systems such as PKI, XKMS, GPGinto the RDF world by interfacing to the existing systems and converting into and out of RDF.

It would not be a goal to develop a specific ontology for policy as a standard or recommended one -- we assume that policy systems will take much time to develop.

A goal would be to test the proof-checking techniques. Cwm's --why flag would produce a proof of its current knoweldge base, which a proof validator could verify, regenerating the knowledge base. This code has (2004) rotted in that it wasnot kept up to date with advances in the platform.
